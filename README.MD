# Onix.Framework

<b>Descrição</b><br>
Framework base para projetos .NET, criado para padronizar arquitetura, abstrações e reutilização de código entre múltiplos sistemas.

<br>

<b>Objetivo</b><br>
Fornecer uma base sólida e consistente para o desenvolvimento de aplicações modulares, facilitando manutenção, evolução e escalabilidade.

<br>

<b>Principais Recursos</b><br>
<b>- Classes base reutilizáveis</b><br>
Entity, AggregateRoot, ValueObject<br><br>

<b>- Abstrações comuns</b><br>
Repositórios, Unit of Work, contratos e notificações<br><br>

<b>- Padrões arquiteturais</b><br>
DDD, Onion Architecture, Clean Architecture<br><br>

<b>- Padronização entre projetos</b><br>
Convenções de organização, nomenclatura e separação de camadas

<br>

<b>Uso</b><br>
<b>1 - Adicionar o Onix.Framework ao projeto</b><br>
Referencie o projeto ou pacote do Onix.Framework na solução desejada.

<br><br>

<b>2 - Utilizar como camada base</b><br>
O framework deve ser usado como base de domínio e abstrações, servindo de fundação para APIs, serviços e módulos do sistema.

<br><br>

<b>Benefícios</b><br>
<b>- Redução de código duplicado</b><br>
<b>- Padronização arquitetural</b><br>
<b>- Aumento de produtividade</b><br>
<b>- Facilidade na criação de novos módulos</b><br>
